# Cours – Infrastructure de Données Géographiques (IDG)

## Définition
Une **Infrastructure de Données Géographiques (IDG)** est une organisation reposant sur :
- des **accords de partage**
- une **coordination entre les membres**
- des **systèmes informatiques**

Ces systèmes intègrent plusieurs services :
- catalogues
- serveurs
- logiciels
- données
- applications
- pages web

Ils permettent la **gestion et la diffusion de l’information géographique** :
- cartes
- orthophotoplans
- images satellites

---

## Historique
Les **IDG apparaissent dans les années 1990** avec :
- le développement des **SIG**
- l’essor de **l’Internet**

Objectif :
- faciliter l’accès aux données
- échanger les informations
- partager les données géographiques

---

## Approches des IDG

### Approche par composantes
Décrit les éléments techniques :
- données
- métadonnées
- services
- logiciels
- acteurs

# Approche réseau
Considère l’IDG comme **un réseau d’acteurs qui coopèrent pour partager les données**.

---

## Typologies des IDG

### IDG généraliste (territoriale)
Diffuse des données **multi-thématiques liées à un territoire**.

Exemples :
- région
- département
- pays

## IDG thématique
Se concentre sur **un thème spécifique**.

Exemples :
- littoral
- risques
- santé

---

## Services INSPIRE

- **Service de recherche** → trouver les données (CSW)
- **Service de consultation** → visualiser les données (WMS)
- **Service de téléchargement** → récupérer les données (WFS)
- **Service de transformation** → adapter les données aux standards
- **Service d’appel** → exécuter un traitement

## Standards et Normes

### Norme
Règle officielle définie par un organisme de normalisation.

Exemple :
- ISO

### Standard
Spécification technique permettant l’interopérabilité.

Exemples :
- WMS
- WFS
- CSW

## Interopérabilité

Permet aux systèmes d’échanger des données.

### Champ sémantique
Définit :
- le sens
- la structure des informations

### Champ géographique
Définit :
- le système géodésique
- la projection cartographique

Exemples :
- WGS84
- Lambert 93

### Champ informatique
Concerne les échanges entre systèmes :

- protocoles réseau
- logiciels
- bases de données

## Principes FAIR

**F – Findable**  
Les données doivent être faciles à trouver.

**A – Accessible**  
Les données doivent être accessibles.

**I – Interoperable**  
Les données doivent être compatibles avec d’autres systèmes.

**R – Reusable**  
Les données doivent être réutilisables.

## Approches techniques

### SaaS
Solution hébergée dans le **cloud par un fournisseur**.

### Hybride
Combinaison :
- cloud
- infrastructure interne

### Auto-gérée

-  Internalisation
Infrastructure gérée **entièrement en interne**.

- Hébergement délégué
Serveur externe mais **gestion interne**.

-  Externalisation
Le prestataire **gère toute l’infrastructure**.
